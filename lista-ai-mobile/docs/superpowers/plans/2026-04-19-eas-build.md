# EAS Build — Multi-Environment CI/CD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Automate Android + iOS builds for `test` (develop branch) and `prod` (version tags, approval-gated) environments using EAS Build and GitHub Actions.

**Architecture:** Two EAS build profiles (`preview` / `production`) in `eas.json` with separate app IDs and name overrides. A single GitHub Actions workflow (`build.yml`) with two independent jobs, each scoped to its own trigger and secret set. Prod job uses a GitHub `production` environment to enforce manual approval before EAS is invoked.

**Tech Stack:** EAS Build (Expo Application Services), GitHub Actions, `expo/expo-github-action@v8`, `eas-cli ≥ 16`

## Task 1: Create `eas.json`

**Files:**
- Create: `lista-ai-mobile/eas.json`

- [ ] **Step 1: Create the file**

```json
{
  "cli": {
    "version": ">= 16.0.0"
  },
  "build": {
    "preview": {
      "android": {
        "buildType": "apk",
        "packageName": "com.listai.app.test",
        "applicationId": "com.listai.app.test"
      },
      "ios": {
        "simulator": false,
        "bundleIdentifier": "com.listai.app.test"
      },
      "env": {
        "EXPO_PUBLIC_APP_NAME": "Lista AI (Test)"
      }
    },
    "production": {
      "android": {
        "buildType": "app-bundle",
        "packageName": "com.listai.app",
        "applicationId": "com.listai.app"
      },
      "ios": {
        "bundleIdentifier": "com.listai.app"
      },
      "env": {
        "EXPO_PUBLIC_APP_NAME": "Lista AI"
      }
    }
  }
}
```

> **Note on app IDs:** `com.listai.app` is a placeholder. Replace with your actual reverse-domain identifier before the first EAS build. The `.test` suffix on the preview profile is what allows both apps to coexist on a device.

> **Note on env vars:** `EXPO_PUBLIC_API_BASE_URL` and the Google OAuth client IDs are NOT in this file — they are secrets injected by GitHub Actions at build time via `--env` flags (see Task 2). Only the non-secret, profile-specific `EXPO_PUBLIC_APP_NAME` lives here.

## Task 2: Create GitHub Actions build workflow

**Files:**
- Create: `.github/workflows/build.yml`

- [ ] **Step 1: Create the workflow file**

```yaml
name: EAS Build

on:
  push:
    branches:
      - develop
    tags:
      - 'v*.*.*'

jobs:
  build-test:
    name: Build (test)
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/develop'
    defaults:
      run:
        working-directory: lista-ai-mobile
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
          cache-dependency-path: lista-ai-mobile/package-lock.json

      - name: Install dependencies
        run: npm ci

      - uses: expo/expo-github-action@v8
        with:
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}

      - name: EAS Build (preview)
        run: |
          eas build \
            --profile preview \
            --platform all \
            --non-interactive \
            --env "EXPO_PUBLIC_API_BASE_URL=${{ secrets.TEST_API_URL }}" \
            --env "EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID=${{ secrets.TEST_GOOGLE_WEB_CLIENT_ID }}" \
            --env "EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID=${{ secrets.TEST_GOOGLE_IOS_CLIENT_ID }}" \
            --env "EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID=${{ secrets.TEST_GOOGLE_ANDROID_CLIENT_ID }}"

  build-prod:
    name: Build (production)
    runs-on: ubuntu-latest
    if: startsWith(github.ref, 'refs/tags/v')
    environment: production
    defaults:
      run:
        working-directory: lista-ai-mobile
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
          cache-dependency-path: lista-ai-mobile/package-lock.json

      - name: Install dependencies
        run: npm ci

      - uses: expo/expo-github-action@v8
        with:
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}

      - name: EAS Build (production)
        run: |
          eas build \
            --profile production \
            --platform all \
            --non-interactive \
            --env "EXPO_PUBLIC_API_BASE_URL=${{ secrets.PROD_API_URL }}" \
            --env "EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID=${{ secrets.PROD_GOOGLE_WEB_CLIENT_ID }}" \
            --env "EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID=${{ secrets.PROD_GOOGLE_IOS_CLIENT_ID }}" \
            --env "EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID=${{ secrets.PROD_GOOGLE_ANDROID_CLIENT_ID }}"
```

## Task 3: Configure GitHub repository (manual steps)

These steps must be done in the GitHub web UI — they cannot be scripted.

- [ ] **Step 1: Create the `production` environment**

  Go to: **GitHub repo → Settings → Environments → New environment**
  - Name: `production`
  - Enable **Required reviewers**, add yourself (or your team)
  - Save

- [ ] **Step 2: Add repository-level secrets**

  Go to: **GitHub repo → Settings → Secrets and variables → Actions → New repository secret**

  Add each of the following:

  | Name | Value |
  |---|---|
  | `EXPO_TOKEN` | Token from expo.dev → Account Settings → Access Tokens |
  | `TEST_API_URL` | e.g. `https://test-api.yourdomain.com` |
  | `TEST_GOOGLE_WEB_CLIENT_ID` | From Google Cloud Console (test project) |
  | `TEST_GOOGLE_IOS_CLIENT_ID` | From Google Cloud Console (test project) |
  | `TEST_GOOGLE_ANDROID_CLIENT_ID` | From Google Cloud Console (test project) |

- [ ] **Step 3: Add `production` environment secrets**

  Go to: **GitHub repo → Settings → Environments → production → Add secret**

  Add each of the following:

  | Name | Value |
  |---|---|
  | `PROD_API_URL` | e.g. `https://api.yourdomain.com` |
  | `PROD_GOOGLE_WEB_CLIENT_ID` | From Google Cloud Console (prod project) |
  | `PROD_GOOGLE_IOS_CLIENT_ID` | From Google Cloud Console (prod project) |
  | `PROD_GOOGLE_ANDROID_CLIENT_ID` | From Google Cloud Console (prod project) |

- [ ] **Step 4: Initialize EAS project (first time only)**

  Run locally from `lista-ai-mobile/`:

  ```bash
  cd lista-ai-mobile
  npx eas-cli init
  ```

  This links the project to your Expo account and writes a `projectId` into `app.json`. Commit the updated `app.json` afterwards:

  ```bash
  git add lista-ai-mobile/app.json
  git commit -m "chore: link EAS project ID"
  ```

## Task 4: Smoke test the test build

- [ ] **Step 1: Push a commit to `develop` and watch the run**

  ```bash
  git checkout -b develop
  git push -u origin develop
  ```

  Go to **GitHub → Actions → EAS Build** — the `build-test` job should start. The `build-prod` job should NOT start (wrong trigger).

- [ ] **Step 2: Verify EAS dashboard shows a running build**

  Go to [expo.dev](https://expo.dev) → your project → Builds. Confirm a `preview` profile build is running for both Android and iOS.

- [ ] **Step 3: Verify approval gate on prod**

  Push a version tag:

  ```bash
  git tag v0.1.0
  git push origin v0.1.0
  ```

  In **GitHub → Actions → EAS Build**, the `build-prod` job should appear with a **"Waiting for approval"** badge. It must NOT proceed until you approve it in the UI.

  Approve it, then confirm the EAS dashboard shows a `production` profile build starting.
